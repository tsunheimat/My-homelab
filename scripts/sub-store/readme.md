Sub-Store 節點重新命名與清洗腳本
====================

這是一個 Sub-Store 腳本操作（Script Operator），用於將雜亂無章的代理節點名稱標準化。它會自動識別國家/地區，可選加入國旗，保留關鍵線路資訊（如 IPLC、倍率、自訂關鍵字），並按腳本內建規則重新編號輸出。

✨ 主要功能
------

* **多模式辨識**：自動辨識中文、英文全名、英文縮寫、國旗圖示等多種格式的節點名稱。
* **格式統一**：將節點統一重新命名為 `國家名 + 序號`，可配合 `flag`、`name`、`nf`、`fgf`、`sn` 組成自訂格式（例如：`香港 01`、`🇭🇰 香港 01` 或 `🇺🇸 HK 01`）。
* **智慧保留**：支援保留特定關鍵字（如 IPLC、IEPL）、倍率（1.5x）、以及自訂的線路識別。
* **進階資訊提取**：支援提取並保留香港 ISP（如 PCCW, HKT）、中國地區與中轉資訊，以及加拿大城市等進階區域資訊。
* **關鍵字替換**：支援在保留關鍵字的同時進行重新命名（例如將 `GPT` 替換為 `OpenAI`）。
* **垃圾清洗**：按內建清理詞過濾包含「到期」、「剩余」、「官方」、「订阅」、「流量」等無效節點。
* **自動去重與排序**：自動為同名節點新增序號，並可用 `blpx` 對特殊線路（如倍率、IPLC）集中排序；目前邏輯是普通節點在前、特殊節點在後。

🚀 使用方法
-------

在 Sub-Store 的 **腳本操作 (Script Operator)** 中新增此腳本的 URL，並在 URL 後透過 `#` 新增參數。

**基本語法：**

```
https://raw.githubusercontent.com/tsunheimat/My-homelab/refs/heads/main/scripts/sub-store/rename.js#參數1&參數2=值&參數3
```

* * *

⚙️ 參數詳解
-------

所有參數均透過 URL Hash 傳遞，多個參數以 `&` 連接。

### 1\. 輸入與輸出控制 (基礎)

| 參數 | 說明 | 範例 |
| --- | --- | --- |
| **`in`** | **輸入格式指定**<br><br> <br><br>預設為自動判斷，順序為中文、國旗、英文全名、英文簡稱。若識別不準，可強制指定原始名稱的類型。 <br><br> <br><br>可選值：`zh`/`cn`(中文), `us`/`en`(英文簡稱), `quan`(英文全名), `flag`/`gq`(國旗)。 | `#in=us` |
| **`out`** | **輸出格式指定**<br><br> <br><br>重新命名後的國家名格式。 **預設為中文**（腳本內建名稱為簡體，例如 `美国`、`中国`）。 <br><br> <br><br>可選值：`zh`/`cn`(中文), `us`/`en`(英文簡稱), `quan`(英文全名), `flag`/`gq`(僅限國旗)。 | `#out=us` (輸出 HK, US) |
| **`nm`** | **保留未符合節點**<br><br> <br><br>預設情況下，無法辨識國家的節點會被刪除。新增此參數可保留原名，但仍會套用 `name`/`fgf` 前綴處理與最後的自動序號。 | `#nm` |

### 2\. 格式與美化 (前綴/後綴/國旗)

| 參數 | 說明 | 範例 |
| --- | --- | --- |
| **`flag`** | **新增國旗**<br><br> <br><br>在節點名稱中加入對應的 emoji 國旗。預設順序下，國旗在 `name` 前綴之前；配合 `nf` 可把 `name` 放到國旗前面。 | `#flag` |
| **`fgf`** | **分隔符號**<br><br> <br><br>設定前綴、國旗、倍率、國家名稱、保留資訊等主要片段之間的分隔符號。預設為空格。 | `#fgf=-` |
| **`sn`** | **序號分隔符號**<br><br> <br><br>設定國家名稱與序號（01, 02）之間的分隔符號。預設為空格。 | `#sn=_` |
| **`name`** | **新增自訂前綴**<br><br> <br><br>新增自訂名稱（如機場名稱）。預設會放在國旗後、國家名稱前；若未啟用 `flag`，則等同放在最前方。 | `#name=我的機場` |
| **`nf`** | **前綴優先**<br><br> <br><br>配合 `name` 使用，將自訂前綴放在國旗前面。 | `#nf` |
| **`entry`** | **保留同機入口名**<br><br> <br><br>用於同一台主機有多個入口 / 出口組合的節點。指定主機短名後，腳本會從原名、`server`、`sni` 匹配入口。若原名已經有完整入口名，例如 `arm1-1` 或 `arm1-1-v6`，會直接保留；若只有 `direct-v4 01` 這類舊格式，才會轉成 `arm1-1 direct-v4`。 | `#entry=arm1&one` |
| **`entrypos`** | **入口名位置**<br><br> <br><br>預設入口名放在國家後面，例如 `新加坡 arm1-1 direct-v4`。如需把入口名放最前，可設定 `front`。 | `#entry=arm1&entrypos=front` |
| **`one`** | **單節點去零**<br><br> <br><br>如果某地區只有一個節點，則不顯示序號（去掉 "01"）。 | `#one` |

### 3\. 過濾與關鍵字保留 (核心)

| 參數 | 說明 | 範例 |
| --- | --- | --- |
| **`blkey`** | **保留關鍵字**<br><br> <br><br>擷取並保留原名中的特定字元，大小寫敏感。多個字用 `+` 連接。 <br><br> <br><br>支援重新命名：`舊名>新名`。 | `#blkey=IPLC+GPT>AI` |
| **`blgd`** | **保留固定標識**<br><br> <br><br>開啟內建正規庫，保留 `IPLC`, `IEPL`, `家寬`, `Game` 等常見標識。 | `#blgd` |
| **`bl`** | **保留倍率**<br><br> <br><br>自動正規匹配並保留節點中的非 1 倍倍率（如 `1.5x`, `3倍`），輸出為 `1.5×` 這類格式。 | `#bl` |
| **`nx`** | **保留普通倍率節點**<br><br> <br><br>過濾掉符合內建倍率/高倍正規的節點，用於保留 1 倍或未標示倍率的普通節點。 | `#nx` |
| **`blnx`** | **只保留高倍率標識節點**<br><br> <br><br>只保留名稱中符合內建高倍率正規的節點。依目前程式碼，主要匹配 `高倍`、`2x`/`2倍`（含連續 `2`）、`ˣ²`、`ˣ³`、`ˣ⁴`、`ˣ⁵`、`ˣ¹⁰`。 | `#blnx` |
| **`clear`** | **清理垃圾節點**<br><br> <br><br>過濾包含內建清理詞的節點，如「套餐」、「到期」、「剩余」、「官方」、「客服」、「订阅」、「流量」及英文 `USE`/`USED`/`TOTAL` 等。注意詞庫以簡體中文與英文為主。 | `#clear` |
| **`hkisp`** | **保留香港 ISP**<br><br> <br><br>在香港節點名稱中提取並保留常見 ISP 名稱（如 HKBN, PCCW, HKT 等）。 | `#hkisp` |
| **`cninfo`** | **保留中國資訊**<br><br> <br><br>在中國節點名稱中提取並保留城市名稱與中轉資訊（如南亞中轉）。 | `#cninfo` |
| **`chincanada`** | **保留中/加位置資訊**<br><br> <br><br>同時保留中國節點的地區/中轉資訊，以及加拿大節點的城市資訊。 | `#chincanada` |
| **`blockquic`** | **QUIC 控制**<br><br> <br><br>強行開啟或關閉節點的 QUIC 設定。未設定或不是 `on`/`off` 時，腳本會刪除節點既有的 `block-quic` 欄位。 <br><br> <br><br>可選值：`on`, `off`。 | `#blockquic=off` |

### 4\. 排序

| 參數 | 說明 | 範例 |
| --- | --- | --- |
| **`blpx`** | **特殊節點排序**<br><br> <br><br>對符合內建特殊正規的節點（倍率、IPLC、IEPL、Game 等）排序並集中到清單後段；普通節點保持在前。 | `#blpx` |

📝 設定範例
-------

以下是幾種常見的配置場景組合：

#### 場景 1：極簡模式（中文 + 自動序號）

將所有節點重新命名為“香港 01”，“美國 01”等。

```
#clear
```

#### 場景 2：英文簡寫 + 國旗 + 保留倍率 + 保留專線標識

_輸入：_ `[機場] Hong Kong IPLC 專線 1.5x` _輸出：_ `🇭🇰 1.5× HK IPLC 01`

```
#out=us&flag&bl&blkey=IPLC
```

#### 場景 3：自訂前綴 + 取代關鍵字 + 特殊符號分隔

_輸入：_ `US_GPT_Line_05` _輸出：_ `MyProxy-🇺🇸-美国-OpenAI-01` _邏輯：_ 新增前綴 MyProxy，分隔符號 `-`，將 GPT 改名為 OpenAI，輸出中文國家名稱，並由腳本重新從 `01` 編號。

```
#name=MyProxy&nf&flag&fgf=-&sn=-&blkey=GPT>OpenAI
```

#### 場景 4：只保留進階節點並排序

只保留符合目前 `blnx` 正規的高倍率標識節點，並對保留下來的特殊標識節點集中排序到清單後段。

```
#blnx&blpx&blgd
```

#### 場景 5：同一台主機保留入口名與出口名

_輸入：_ `oracle 新加坡 arm1 direct-v4 01` _輸出：_ `新加坡 arm1-1 direct-v4`
_輸入：_ `oracle 新加坡 arm1-1-v6 warp-v6 02` _輸出：_ `新加坡 arm1-1-v6 warp-v6`

```
#entry=arm1&one
```

⚠️ 注意事項
-------

1. **執行順序**：腳本會先做 `clear`/`nx`/`blnx`/`key` 過濾，接著進行地名預處理與 `block-quic` 處理，再提取 `blkey`/`blgd`/`bl`/`hkisp`/`cninfo` 等保留資訊，最後組合新名字、重新編號，並依 `one`/`blpx` 做後處理。
2. **國旗識別**：如果使用了 `in=flag`，腳本會嘗試讀取原名中的 emoji 國旗來判斷國家，請確保不要在腳本操作前移除國旗。
3. **正規匹配**：腳本內建了龐大的國家/城市匹配庫（包含中文、英文、常用別名如「獅城」、「大坂」等），一般情況下不需要手動指定 `in` 參數；但中文輸出名稱來自腳本內建簡體清單。
